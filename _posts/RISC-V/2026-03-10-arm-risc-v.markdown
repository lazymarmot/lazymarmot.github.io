# ARM 과 RISC-V

ARM = ‘완성품 중심 RISC’

RISC-V = ‘설계도 중심 RISC’

같은 RISC철학이지만 누가 주도권을 쥐고 있느냐가 다름

![image.png](/assets/posts_image/RISC-V/ARM_RISC-V/image1.png)

누가 통제하느냐 (제일 본질적인 차이)

| ARM | RISC-V |
| --- | --- |
| ISA 설계 : ARM<br>코어 설계 : ARM<br>확장 방향 : ARM이 결정<br><br>ARM은 ARM이 모든 큰 결정을 함 | ISA : 공개<br>코어 설계 : 각 회사 마음<br>확장 : 네가 정의<br><br>RISC-V는 RISC-V International에서 규칙만 관리함 |

<br>

### ARM ISA

- 명령어 많음
    - addressing mode 다양
    - SIMD, system 명령 강력
    - backward compatibility 엄청 중요
    
    ARM은:
    
    ```c
    LDP x0, x1, [x2], #16
    
    -> load + 주소 증가까지 한 번에
    → 성능 좋고 코드 밀도 높음
    → 대신 디코더 & 검증이 빡셈
    ```
    
<br>

### RISC-V ISA

- 기본 명령어 **아주 최소**
    - 확장은 모듈식
    - 불필요한 기능 없음
    
    RISC-V는:
    
    ```c
    lw x1, 0(x2)
    addi x2, x2, 4
    
    -> “한 명령 = 한 일”
    -> 하드웨어 설계자 친화적
    ```
    

확장 방식 차이

| 항목 | ARM | RISC-V |
| --- | --- | --- |
| 확장 주도 | ARM | 사용자 |
| 커스텀 명령 | 거의 불가 | 매우 쉬움 |
| AI/가속기 | ARM 제공 범위 | 회사 전용 가능 |

- ARM: NEON / SVE 범위 안에서 최적화
- RISC-V: `X_ai_dotprod` 같은 **자체 명령 추가**
    
    ⇒ 그래서 AI·반도체 스타트업은 RISC-V로 감
