# RISC와 CISC

![image.png](/assets/posts_image/RISC-V/RISC_CISC/image1.png)

### CISC (Complix Instruction Set Computer)

- 명령어 복잡함
- 명령어 종류 많고, 길이도 제각각
- 한 명령어가 여러가지 일을 한번에 함
(메모리 접근 + 연산 + 저장을 한 명령어에서 처리)
- “겉으로는 명령어 1개 지만 속으로는 미니 프로그램(마이크로코드)이 돌아간다”
- 마이크로코드 ( =순서표)
    - 복잡한 명령은 내부적으로 단순한 동작 여러 개로 나눠 순서대로 실행한다.
    
    ```c
    ADD [MEM], R1
    
    -> "메모리에서 값 읽어서 레지스터 R1이랑 더한 뒤 다시 메모리에 저장"
    ```
    
    CPU 내부에서 하는 일 (마이크로코드)
    
    ```c
    1. MAR ← MEM 주소
    2. MDR ← Memory[MAR]        ; 메모리 읽기
    3. ALU_OUT ← MDR + R1       ; ALU 연산
    4. Memory[MAR] ← ALU_OUT   ; 메모리에 다시 저장
    
    -> 이 각각이 마이크로 연산 (micro-operation)
    -> 이 순서를 적어둔 게 마이크로코드
    ```
    
    “마이크로코드는 하드웨어를 제어하는 펌웨어”
    
    | 관점 | 설명 |
    | --- | --- |
    | 소프트웨어 같음 | 명령 순서가 “프로그램”처럼 저장됨 |
    | 하드웨어 같음 | CPU 안 ROM에 들어 있음 |

### RISC (Reduced Instruction Set Computer)

- 고정된 길이 명령어 사용
- 명령어 개수도 적음 / 길이 거의 고정
- 레지스터 사이의 연산만 실행
- 메모리 접근은 load/store등 명령어 몇 개로 제한함

```c
ADD R1, R2, R3

-> R2 + R3 → R1
-> 메모리 접근 ❌
-> 딱 ALU 연산 하나
-> 사이클에 끝내도록 설계
```

```c
CPU 내부:

ALU_OUT ← R2 + R3
R1 ← ALU_OUT

-> 굳이 마이크로코드로 나눌 필요 없음
```

“명령어 자체가 이미 마이크로 연산 수준”

CISC / RISC(마이크로코드 관점)

| 구분 | CISC | RISC |
| --- | --- | --- |
| 명령어 복잡도 | 매우 복잡 | 매우 단순 |
| 명령어 길이 | 가변 | 고정 |
| 메모리 접근 | 명령어 안에서 가능 | Load / Store 분리 |
| 실행 방식 | 마이크로코드로 분해 | 하드와이어드 |
| 파이프라인 | 어려움 | 매우 쉬움 |
| 예시 | x86 | ARM, RISC-V |

작업:

메모리에서 값 읽어서 +1 하고 다시 저장

CISC (x86느낌)

```c
INC [0x1000]

이 한줄이 내부적으로
- 메모리 읽고
- +1 연산
- 다시 메모리에 씀

한 명령 = 여러단계
```

RISC 

```c
LOAD r1, [0x1000]
ADD  r1, r1, #1
STORE r1, [0x1000]

명령어는 많지만 각각 하는 일은 하나씩
```

| --- | --- |
| CISC | x86 |
| RISC | ARM, RISC-V, MIPS, Power |
